# Deployment Guide

This guide covers deploying the Movie Recommendation System on various platforms.

## Streamlit Cloud Deployment

### Important: Pre-deployment Steps

The Streamlit app requires preprocessed pickle files that are generated by running the recommendation system. These files are too large (>100 MB) to commit to GitHub, so you need to generate them on the deployment platform.

### Option 1: Streamlit Cloud (Recommended)

1. **Fork the repository** on GitHub

2. **Prepare your deployment**:
   - The CSV files also need to be available on the deployment server
   - You can either:
     - Keep CSV files in the repository (larger repo)
     - Use the deployment as a trigger to generate fresh pickle files
     - Store CSV files in cloud storage

3. **Create a `streamlit_secrets.toml` file** in `.streamlit/` (for local testing)

4. **Deploy on Streamlit Cloud**:
   - Go to https://streamlit.io/cloud
   - Sign in with GitHub
   - Click "New app"
   - Select this repository
   - Set main file to `app.py`

5. **Handle pickle file generation**:
   
   **Option A: Add startup script**
   
   Create `setup_env.py`:
   ```python
   import os
   import subprocess
   
   # Check if pickle files exist
   if not os.path.exists('movies_dict.pkl') or not os.path.exists('similarity.pkl'):
       print("Generating pickle files...")
       subprocess.run(['python', 'recommendation_system.py'])
   ```
   
   Then modify `app.py` to call this first.

   **Option B: Include pre-generated files**
   
   1. Run locally: `python recommendation_system.py`
   2. Commit pickle files (note: GitHub has 100 MB limit)
   3. Push to GitHub

### Option 2: Heroku Deployment

1. **Ensure pickle files exist locally**:
```bash
python recommendation_system.py
```

2. **Create `Procfile`**:
```
web: streamlit run --server.port $PORT app.py
```

3. **Create `setup.sh`**:
```bash
mkdir -p ~/.streamlit/

echo "\
[theme]
primaryColor = '#FF6B6B'
backgroundColor = '#FFFFFF'
secondaryBackgroundColor = '#F0F2F6'
textColor = '#262730'
font = 'sans serif'

[client]
showErrorDetails = true

[logger]
level = 'info'

[server]
maxUploadSize = 200
" > ~/.streamlit/config.toml
echo "done"
```

4. **Deploy**:
```bash
heroku login
heroku create your-app-name
git push heroku main
```

### Option 3: Docker Deployment

Create `Dockerfile`:
```dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

# Generate pickle files on startup
RUN python recommendation_system.py

EXPOSE 8501

CMD ["streamlit", "run", "app.py"]
```

Create `docker-compose.yml`:
```yaml
version: '3.8'

services:
  streamlit:
    build: .
    ports:
      - "8501:8501"
    volumes:
      - ./:/app
```

Run locally:
```bash
docker-compose up
```

## Important Notes

### CSV Files
- The TMDB CSV files (tmdb_5000_movies.csv, tmdb_5000_credits.csv) are required
- Download from: https://www.kaggle.com/tmdb/tmdb-movie-metadata
- These should be in the project root or the path updated in `recommendation_system.py`

### Pickle Files
- **Size**: similarity.pkl is ~176 MB, movies_dict.pkl is ~2.2 MB
- **Generation time**: Usually 3-5 minutes depending on system
- **Storage**: Not committed to GitHub due to size limits
- **On deployment platforms**: Need to be generated during app initialization or before deployment

### Memory Requirements
- The similarity matrix (176 MB) consumes significant memory
- Ensure your server has at least 512 MB RAM
- Streamlit Cloud free tier should handle this

## Troubleshooting

### "Pickle files not found" Error
**Solution**: Run `python recommendation_system.py` in your deployment environment

### Out of Memory
**Solution**: 
- Upgrade your server plan
- Consider using a smaller similarity matrix optimization

### Slow Load Times
**Solution**: The initial data loading can take time. This is normal for the first access. Streamlit caches the data, so subsequent requests are faster.

## Monitoring Deployment

Once deployed, monitor:
- App loads and displays the movie selection interface
- Movie selection dropdown populates correctly
- Recommendations generate without errors
- Performance and memory usage (especially important for large datasets)
